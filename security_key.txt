In a high-security vault system, access keys need to be periodically updated for enhanced security. A new key is generated based on the following rules: 1. The new key must be exactly k characters long. 2. The new key can only be made from the characters present in the current key. 3. The new key must be lexicographically larger than the current key. In other words, it should come after the current key in dictionary order. Given the current key, find the smallest possible new key of length k that meets these conditions. Return "-1" if no such key is possible. Note: •  A string p is lexicographically smaller than string q if p comes before q in the dictionary. For example, "abc" is smaller than "abcd", and "abd" is smaller than "abe".
